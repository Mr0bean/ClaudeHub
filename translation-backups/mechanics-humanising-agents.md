# 人性化代理 | ClaudeLog

`自定义代理`通过其专门的角色、系统提示和工具配置被赋予了独特的个性。然而，默认的交互模式可能会感觉过于正式。我们可以通过使用`昵称`和富有表现力的`文字表情`来人性化我们的代理，从而创建更吸引人的协作。

* * *

* * *

## 超越正式调用[​](#beyond-formal-invocation "Direct link to 超越正式调用")

传统的代理调用感觉很机械：

```bash
ask performance optimiser assistant to review the changes

```

使用[昵称](/mechanics/agent-engineering/#agent-nicknaming-for-efficiency)，交互变得更加自然：

```bash
ask P1 to review the changes

```

但我们可以通过添加`文字表情`来增加个性，走得更远，创建感觉更像协作者而非工具的代理。

* * *

* * *

## 文字表情个性[​](#text-face-personalities "Direct link to 文字表情个性")

在探索个性化代理的不同方式时，我产生了探索`文字表情`的想法。当它第一次加载时，我就知道这是个成功！每个表情代表了代理的个性、角色或典型情绪，同时补充了开发者喜爱的简洁终端美学。

## 实施策略[​](#implementation-strategy "Direct link to 实施策略")

在设计自己的人性化助手时，使用上述类别作为起点：

1.  **将个性与功能相匹配** - `┗(▀̿Ĺ̯▀̿ ̿)┓ Git管理器`需要与`(• ε •) 温柔的重构器`不同的能量
2.  **选择适当的强度** - 比较轻松的`¯\_(ツ)_/¯ 休闲代码审查员`与激烈的`┌༼◉ل͟◉༽┐ 语法检查器`
3.  **考虑你的团队环境** - `(┛ಠДಠ)┛彡┻━┻ 沮丧的开发者`可能适合休闲团队，但不适合正式环境
4.  **将复杂性与专业化相匹配** - 像`⋌༼ •̀ ⌂ •́ ༽⋋ 算法专家`这样的复杂表情应该匹配同样专业的角色
5.  **测试显示兼容性** - 确保你选择的`文字表情`在不同终端和系统中正确渲染
